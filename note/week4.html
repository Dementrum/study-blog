День 1. Хелперы

Блок с кодом

<dl>
 <dt>Чакушка</dt>
  <dd>Используется многими продвинутыми товариСЧами для утоления жажды(спрайт отдыхает).</dd>
 <dt>Флян</dt>
  <dd>Используется теми же продвинутыми товариСЧами, если чакушка не спасла от жажды.</dd>
 <dt>Канистра</dt>
  <dd>Используется в крайних случаях при затяжных праздниках.</dd>
</dl>

Блок Важно:

Никогда не размещайте в тегах абзаца другие абзацы. Так как у вас будет ошибка валидации кода.

Да и если подумать логически - разве может быть абзац в абзаце?

Блок Фишка:

Если раньше в тег ссылки можно было добавлять только один другой тег, то сейчас вы можете добавлять сразу несколько.

Не беспокойтесь, ошибок валидации не будет!

День 2. Аудио

Музыкальный плеер на сайте

Тег <audio> позволяет проигрывать музыкальные композиции на сайте. Путь к странице можно задать двумя способами. Через тег <source>, либо через атрибут src

<audio src="путь к картинке"></audio>
<audio>source src="путь к картинке"</audio>

Атрибуты музыкального проигрывателя:

controls - указывает, что необходимо отображать элементы контроля над плеером.

autoplay - задаёт автоматическое воспроизведение файла.

loop - зацикливает воспроизведение файла.

preload - указывает браузеру, как надо подгружать мультимедиа на странице.

  • none - не загружать файл.
  • auto - стоит по умолчанию, файлы загружаются вместе со страницей.
  • metadata - браузер подгружает только информацию о файле.

День 3. Видео.

Видеопроигрыватель на сайте:

Тег <video> позволяет проигрывать видео композиции на сайте. Путь к странице можно задать двумя способами. Через тег <source>, либо через атрибут src

<video src="путь к картинке"></video>
<video>source src="путь к картинке"</video>

Атрибуты, которые мы рассмотрели к аудипроигрывателю, подходят также и к видеопроигрывателю. Но у него есть ещё несколько дополнительных атрибутов.

Атрибуты видеопроигрывателя:

width - указывает ширину для видео в пикселях или процентах.

height - указывает высоту для видео в пикселях или процентах.

poster - вставляет постер, (картинку заставку) в видео, которая отображается до воспроизведения.

Помните, что пропорции постера и видео должны соответствовать, иначе постер будет отображаться не пропорционально и не красиво. Будет искажаться.

Тег <source> - указывает путь к файлу, который необходимо проиграть. Ниже рассмотрим атрибуты этого тега.

  • type - указывает тип воспроизводимого файла.
  • src - задает путь к файлу.
  • media - для какого устройства должен производиться этот файл.
  • all - для всех устройств.
  • handheld - для смарфонов, телефонов и иже с ними.
  • screen - для экранов монитора.
  • speech - для речевых синтезаторов.
    • .
  • projection - для проекторов.
  • tv - для телевизоров.
  • tty - для телетайпов.
  • print - для печати.

День 4. Формы.

Формы задаются при помощи тега <form> который является парным. содержимое формы состоит на 99% из тегов <input> различного типа. Этот тег является одиночным.

<form action="#">accept-charset="utf-8" method="post" name="test"
    <input name="name">
    <input name="mail">
    <input name="text">
</form>

Атрибуты форм:

  • action - указывает на адрес программы или файла обработчика.
  • accept-charset - устанавливает кодировку отправленных данных.
  • method - имеет два значения(get или post). Указывает, как передавать данные.
  • name - задает уникальное имя для формы. Обязательный атрибут для формы.
  • enctype - указывает способ кодировки отправленных данных.
  • target - указывает каким образом будет отображен результат выполнения формы.
  • autocomplete - включает/выключает (on/off) автозаполнение форм.
  • novalidate - отменяет проверку на правильность заполнения полей форм.
Помните, что имя формы и имена её элементов должны быть уникальными. Имена задаются произвольно, они могут состоять из латинских букв, цифр и дефисов.

Почти все элементы формы задаются через тег <input>, и без этого тега форма работать не сможет.

У тега input обязательно должны присутствовать два атрибута - name и type.

День 5. Тег input

В HTML5 появилось множество новых полей. Будьте внимательны - некоторые новые поля ещё некорректно работают в браузерах.

Новые значения атрибута TYPE в HTML5

Пример формы

  • color - поле для ввода цвета.
  • email - указывает, что в поле должен введен адрес электронной почты.
  • number - числовое поле.
  • date - указывает, что в этом поле содержится дата.
  • month - указывает месяц.
  • week - указывает неделю.
  • datetime - указывает дату и время.
  • datetime-local - указывает местное время и дату.
  • time - указывает время.
  • url - для ввода адресной строки.
  • tel - поле для телефона.
  • search - поле для поиска информации.
  • range - задает ползунок.

Будьте внимательны, так как не все типы полей поддерживабтся современными браузерами. Обычно, если поле не поддерживается, оно обрабатывается браузером, как обычное текстовое поле.

Поля email, number, range, seatch, tel и url уже поддерживаются всеми современными браузерами. И вы можете использовать эти типы полей на своих сайтах. Не беспокоясь за работоспособность.
