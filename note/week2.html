День 1. А что такое CSS?

CSS - (стили или таблицы стилей) - используются для того ,чтобы управлять дизайном сайта и оформлением каждого элемента страницы.

Способы заведения стилей:

Связанные стили
Они находятся внутри отдельного файла стилей, и связываются со всеми страницами сайта при помощи специального тега <link>
Глобальные стили
Они находятся на одной странице с HTML и задаются в теге <head>
Внутренние стили
Задаются внутри тега при помощи специального атрибута style.

Этот текст мы сделали красным при помощи внутренних стилей, задав следующий атрибут: style="color:red;

Глобальные стили задаются при помощи тега <style> </style>. Он сообщает браузеру,что внутри него идет CSS

Если у вас сайт состоит из множества страниц - лучше всего подойдут связанные стили. Гдобальные стили - используются на одностраничниках.

Важно: У каждого способа задания стилей есть свой приоритет. Самый высокий приоритет у внутренних стилей. Затем идут глобальные стили. И самый низкий приоритет у связанных стилей.

В CSS ,как и в HTML есть свои комментарии. Они задаются при помощи:

/*текст*/

Синтаксис CSS

Как HTML состоит из тегов, так и CSS состоит из селекторов. селекторы указывают к какому элементу или элементам сайта мы хотим применить стили.

В свою очередь, каждый селектор состоит из свойств, а каждое свойство состоит из значений. Вот базовый синтаксис:

селектор{свойство: значение;}

Пример использования:

body {color: #272727;}

У каждого селектора может быть несколько свойств - ограничений по количеству нет. Свойства задаются через точку с запятой.

селектор{свойство: значение; свойство 2: значение; свойство 3: значение; свойство 4: значение;.....}

Пример использования:

body {color: #272727; background: #efefef;}

И у каждого свойства может быть несколько значений. Значения указываются через пробел.

селектор{свойство: значение значение2 значение3 значение4;}

Пример использования:

body {color: #272727; background: #efefef url("images/bg.png");}

А читабельнее так: body {   color: #272727;   background: #efefef url("images/bg.png"); } Но это уже дело вкуса :)

День 2. Cелекторы

Они используются для того ,чтобы конкретно указать элемент сайта, к которому мы собираемся применить стили.

Для того, чтобы наши гадкие ручонки могли добраться до любого!!! элемента сайта, мы будем использовать различные виды селекторов.

Виды селекторов

Селекторы тегов - используются для того,чтобы конкретно указать теги, к которым мы хотим применить стили. В качестве названий мспользуются имена тегов (без угловых кавычек.)

Пример использования:

h2 {border-bottom: 1px dashed #ccc;}
для всех заголовков 2 уровня задали штрих-пунктирную границу снизу толщиной в 1 пиксель.

Классы(селекторы классов) - нужны для того, чтобы применять стили к отдельным элементам сайта выборочно, а не ко всем сразу.

В файле стилей задаются следующим образом:

.имя класса {стили для него}

В HTML привязываются к тегам при помощи специального атрибута class="имя_класса"

Синий текст Красный текст Черный текст Зеленый текст

Идентификаторы (селекторы идентификаторы) - нужны для того, чтобы применять стили к отдельным элементам сайта выборочно, а не ко всем сразу. Отличаются от классов тем, что они должны быть уникальны и могут использоваться только ОДИН раз на странице.

В файле стилей задаются следующим образом:

#идентификатор {стили для него}

В HTML привязываются к тегам при помощи специального атрибута id="имя_класса"

Синий текст Красный текст Черный текст Зеленый текст

Важно: Имена для идентификаторов и классов могут быть произвольные, однако есть свои ограничения. В именах могут использоваться только латинские буквы, а также знак дефиса и нижнего подчеркивания. Также могут использоваться и цифры. И главное - имя должно начинаться с латинской буквы.

Важно: Помните что у идентификаторов приоритет всегда выше чем у классов

Универсальный селектор - применяет стили сразу для всех элементов сайта.

Выглядит он следующим образом:

*{cтили}

Селекторы атрибутов- позволяют задать стили тегам у которых есть определенные атрибуты

Задаются через квадратные квавычки, в котгорых указывается имя атрибута.

Вот как это выглядит:

[атрибут]{стили}

Эту покрасили в красный цвет через селектор атрибутов.

Фишка: Мы можем дополнительно в селекторе атрибутов указывать значения этих самых атрибутов.

Эту покрасили в зеленый цвет через селектор атрибутов со значением.

День 4. Стройбаны

Два великих тега!

<div> - это пустой тег блочного вида. Который надо использовать для задания структуры сайта.

<span> - это пустой тег строчного вида. Который мы можем использовать для оформления отдельных участков текста.

Блочные элементы - это теги, которые по умолчанию растягиваются на всю ширину родителя. Начинаются с новой строки, и следующий за ним текст тоже переносится

Строчные элементы - это теги, которые не переносят на новую строку.

Важно: Блочные элементы растягиваются на всю ширину родителя. Высота у них по умолчанию зависит от содержимого.

Обтекания

Оно задается при помощи сойства float. Если есть обтекание, то тег, которому мы его задали - перестает переносить элементы после него на новую строку. И эти элементы начинабт его обтекать.

Важно: Если блочному элементу задать обтекание - он перестанет растягиваться на всю ширину родителя.

Значения свойства float:

none - оно присваиваится по умолчанию.

left - оно прижимает элемент к левому краю, и все остальные теги, которыке идут за ним - обтекают его с правой стороны.

right - оно работает наоборот.

Очистка обтекания.

За очистку обтекания отвечает свойство clear. Когда мы его задаем, все теги начиная с тега с этим свойством перестают обтекать.

Значение свойства clear:

none - очистки нет

left - очистка обтекания тегов со свойством float:left;

right - очистка обтекания тегов со свойством float:right;

both - очистка всех обтеканий.

День 5. Измерители

В стилях есть свойства, которые влияют на размер блоков.

Длина и высота

Свойство height - указывает высоту блока.

width - указывает ширину блока.

Важно: Помните, что размеры мы можем указывать только для блочных типов элементов. Для строчных они работать не будут.

Значения и единицы измерения:

auto - задает значения высоты по умолчанию. Ширина блочных элементов зависит от родителя.

% - указывают размер блока в процентах. Размер вычисляется относительно родителя.

pm/in/pt - количественные, точные значения.

Важно: Если картинке прописать только одну какую-то сторону, а для второй указать auto, то она автоматически выровняется согласно пропорциям.

Границы

В первую очередь помните, что любая граница созданная при помощи свойства border влияет дополнительно на размер в зависимости от своей толщины.

border - указывает границу, по умолчанию она отсутствует у многих элементов сайта.

Синтаксис

border: толщина тип цвет;

Толщину можно указывать любую в количественных единицах измерения.

Типы границ:

dotted dashed solid outset inset groove double ridge

Ширина этого блока указана в 300 пикселей. Однако она на самом деле составляет 320 пикселей, так как у этого блока задана граница толщиной в 10 пикселей.

Как указывать границы отдельно:

border-top - граница сверху

border-right - граница справа

border-bottom - граница снизу

border-left - граница слева

У этого блока разные границы, потому что мы использовали специальные стили.

Поля и отступы.

Поля указывают внутренние отступы, а отступы - внешние.

padding - это свойство задает внутренние отступы или поля. Влияет на размер всех элементов, которым оно задано.

Поля можно указывать отдельно:

padding-top - поля сверху

padding-right - поля справа

padding-bottom - поля снизу

padding-left - поля слева

padding - отступы одинаковы со всех сторон

Можно указывать поля разными способами:

padding: верх право низ лево

padding: верх право_лево низ

padding: верх_низ право_лево

Ширина этого блока 300 пикселей, но на самом деле ширина у него 342 пикселя. 40 пикселей за счёт паддингов, и ещё 2 за счет границ.

margin - задает внешние отступы. Они работают точно так же как и поля, за тем исключением,что они не влияют на размер блока.

Важно: Для строчных элементов внешние отступы не работают по вертикали, только справа и слева.
