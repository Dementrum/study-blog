Неделя 9 - Упаковщики

День 1. Введение во Flex.

Flexible Box - переводится, как "гибкий блок".

Flex-контейнер - родительский блок, в котором и будет происходить весь спектакль

Flex-элемент - это дочерние блоки родителя, которые участвуют в нашем спектакле.

Css столбцы columns, float, clear и vertical-align - эти свойства НЕ РАБОТАЮТ, когда мы применяет Flexbox

Flex можно включить, если мы пропишем flex-контейнеру свойство
display: flex;

Не забываем добавлять префиксы:

.flex {
display: -webkit-box;
display: -webkit-flex;
display: -moz-box;
display: -ms-flexbox;
display: flex;
}

Во FlexBox внешние поля (margin) складываются!.

День 2. Задолбали эти оси.

Во Flex-контейнере есть две оси, по которым мы можем управлять положением наших блоков

Главная ось - эта ось, по которой направляются наши блоки.

Поперечная ось - она всегда перпендикулярна главной.

Мы можем поменять направление главной оси при помощи свойства
flex-direction

Значения свойства flex-direction

  • row - наши блоки будут располагаться слева направо-строкой (значение по умолчанию).
  • row-reverse - блоки идут справа налево
  • column - располагает наши блоки столбцом (сверху вниз)
  • column-reverse - располагает блоки наоборот

Выравнивание по осям

justify-content - оно выравнивает блоки по главной оси.

  • flex-start - блоки выравниваются от начала главной оси
  • flex-end - блоки выравниваются с конца главной оси
  • space-between - блоки прижимаются к краям главной оси, а остальные блоки равномерно распределяются.
  • space-around - блоки равномерно распределяются по главной оси
  • center - размещает блоки по центу

align-items - оно выравнивает блоки по поперечной оси.

День 3. Тонкое управление.

Мы можем указать flex- контейнеру, чтобы его блоки переносились на новый ряд.

flex-wrap - управляет переносом блоков на новый ряд.

  • nowrap - блоки расположены в одну линию.
  • wrap - переносит блоки на новую строку или ряд.
  • wrap-reverse - тоже переносит блоки, только в обратном порядке.

Переносы работают только в том случае, если наши блоки не влазят по размерам в один ряд.

align-content - выравнивает ряды блоков по поперечной оси.

  • flex-start - прижимает ряды блоков к началу оси.
  • flex-end -ряды блоков выравниваются с конца главной оси.
  • stretch - оно растягивает ряды блоков по всей поперечной оси.( по умолчанию )
  • center - размещает ряды блоков по центру.
  • space-between -ряды блоков прижимаются к краям главной оси, а остальные блоки равномерно распределяются.
  • space-around - ряды блоков равномерно распределяются по главной оси.

flex-flow - обьединение свойств flex-direction и flex-wrap

День 4. Жадина-говядина.

flex-grow: 0; - указывает, как блок будет занимать свободное пространство внутри контейнера, относительно других блоков.

Иначе, в народе, это свойство называют "жадность" блока

Значение этого свойства, не абсолютное, а относительное. Оно указывает, как блок будет забирать свободное пространство внутри контейнера, относительно других блоков.

flex-shrink:1; - указывает степень сжимаемости блока, если нет свободного пространства.

Жадность работает только со свободным пространством, у сжимаемости -только тогда, когда его не хватает

Помните, что степень сжатия зависит от переизбытка занимаемого пространства, чем оно больше, тем сжатие больше

flex-basis - указывает базовую ширину блока, от которой надо отталкиваться при подсчете жадности и сжимаемости

Значения свойства могут выражаться в пикселях, процентах, em, rem и других величинах

flex: flex-grow flex-shrink flex-basis; - обьединяет вместе все верхние 3 свойства. Идут они в таком же порядке, как мы и разбирали.

День 5. Стройбат батя!.

align-self - позволяет выровнять наши блоки по поперечной оси.

Это свойство, можно назначить каждому отдельному блоку. Оно важнее, чем align-items.

  • flex-start - прижимает блок к началу оси
  • flex-end - блок выравниваются с конца главной оси
  • stretch - оно растягивает блок по всей поперечной оси.
  • center - размещает блок по центру.
  • baseline - выставляет блок по их базовой линии.
  • auto - это значение по-умолчанию означает, что наш блок будет использовать align-items.

order - указывает распорядок блоков.

Использует обычные числа. Могут быть как положительные, так и отрицательные значения.

Значение, по умолчанию у всех блоков 1. То есть, они выводятся как есть в HTML-документе.

Чем меньше значение блока, тем первее он будет выводиться во flex-контейнере(что-то вроде z-index)

Если значения у блоков одинаковые, тогда они выводятся в порядке, в котором прописаны в HTML

Маленькие фишки

Во flex-контейнере свойство margin:auto выравнивает блоки, не только по горизонтали, но и по вертикали.

Чтобы ограничить наши блоки по определенному размеру при свойстве flex-grow, указывайте максимальную высоту и ширину.

Свойство margin: auto криво работает в IE, если дополнительно используется свойства flex-grow и jusity-content.
